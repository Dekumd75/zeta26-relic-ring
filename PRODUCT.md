# Product

## Register

product

## Users

Students, judges, and developers using the Zeta-26 Relic Ring Protocol simulator to explore interplanetary packet routing. They are in a task context: selecting planets, sending messages, inspecting latency, and testing failures.

## Product Purpose

The product simulates packet delivery across a fictional planetary network. Success means a user can quickly see the fastest valid route, understand why it was chosen, inspect per-hop latency and codex conversion, and test how deactivated planets or links change delivery.

## Brand Personality

Technical, atmospheric, precise. The interface should feel like an operational mission console without hiding data behind decoration.

## Anti-references

Avoid marketing landing-page patterns, oversized hero sections, decorative card grids, vague sci-fi ornament, and controls that look dramatic but are hard to scan. Avoid UI that makes routing math feel like a game when users need inspectable calculations.

## Design Principles

- Keep the simulator as the first screen; routing is the product.
- Make state visible: active route, failures, latency, and packet logs should always be easy to locate.
- Prefer dense, structured data over decorative explanation.
- Preserve the space-map identity while keeping controls familiar and fast.
- Treat failures and empty results as first-class states.

## Accessibility & Inclusion

No project-specific accessibility requirements are documented. Default to keyboard-operable controls, visible focus states, readable contrast, and reduced-motion support for canvas and UI animations.
